CI/CD & Monitoring Integration Notes (For Evaluator)

This dashboard represents the single pane of glass for our solution. The underlying system:

  • **CI/CD Integration:** The **Last Deployment Time** is pulled directly from GitHub Actions metadata, ensuring we track changes deployed to the HPA configuration.
  • **Metrics Pipeline:** All data visualized (CPU, Replicas, Latency) flows from Prometheus/Grafana into a dedicated **Monitoring API** which the dashboard consumes.
  • **Traffic Spike Control:** The "Trigger Spike" button is mapped to a Jenkins job that deploys a load generation service (e.g., K6 or Locust) to validate scaling resilience post-deployment.
